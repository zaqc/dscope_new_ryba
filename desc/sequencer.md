# Модуль Sequencer (Последователь циклов зондирования)

Модуль `sequencer` предназначен для высокоточной координации многоканального зондирования и сбора данных. Он управляет временной последовательностью шагов сканирования, распределяет синхроимпульсы запуска и признаки завершения последовательности по всем тактовым доменам проекта, а также контролирует готовность приемных трактов.

---

## 1. Архитектура управления и интеграция с модулем Param

Модуль `sequencer` выполняет роль системного тайминг-контроллера. Он полностью освобожден от хранения настроек коммутации и масок генераторов — эти функции централизованно выполняет модуль параметров `Param`.

1. **Инициализация по глобальному импульсу запуска (`i_sys_sync`):**
   * При поступлении импульса `i_sys_sync` (от модуля `rst_sync` в начале кадра) система производит фиксацию (latch) всех параметров контроля из регистровой карты модуля `Param`.
   * Исполнительные модули `ascan 0..3` и `pulser 0..3` осуществляют сброс внутренних конвейеров и готовятся к новому кадру сканирования.

2. **Пошаговый запуск по импульсам субсинхронизации (`sub_sync`):**
   * Для каждого шага (субцикла) $s \in [0..i\_seq\_count]$ модуль `sequencer` выставляет индекс шага `o_step_index`.
   * Модуль `Param` использует `o_step_index` для мгновенной коммутации аналоговых мультиплексоров ПЭП и обновления масок каналов излучения для модулей `pulser`.
   * По истечении времени стабилизации аналоговых ключей `sequencer` генерирует распределенный импульс запуска `sub_sync` во все тактовые домены.

3. **Определение конца последовательности (`sub_last`):**
   * На последнем шаге кадра (когда $s = i\_seq\_count$) одновременно с импульсами `sub_sync` во все тактовые домены транслируются импульсы `sub_last`.
   * **Применение в `ascan`**: В домене `adc_clk` модуль `ascan` использует совместный приход `o_adc_sub_sync` и `o_adc_sub_last` как индикатор завершения кадра. По окончании регистрации этого финального блока `ascan` переключает страницы двухпортовой памяти буфера («бабочки») и выставляет сигнал готовности данных `o_data_ready` для отправки накопленного кадра на выход.

### Схема синхронизации и CDC-взаимодействия:

```
                      +---------------------------------------+
                      |               sequencer               |
                      |                                       |
    sys_clk --------->| [FSM] ------------------------------->|=====> o_step_index [1:0] (к Param, Rx, Tx)
                      |   |                                   |
    sys_clk --------->|   +---------------------------------->|=====> o_sys_sub_sync / o_sys_sub_last
    adc_clk --------->|   +---> [CDC Pulse Sync] ------------>|=====> o_adc_sub_sync / o_adc_sub_last (к ascan)
    log_clk --------->|   +---> [CDC Pulse Sync] ------------>|=====> o_log_sub_sync / o_log_sub_last (к log-Rx)
    dac_clk --------->|   +---> [CDC Pulse Sync] ------------>|=====> o_dac_sub_sync / o_dac_sub_last (к ВАРУ)
    hi_clk ---------->|   +---> [CDC Pulse Sync] ------------>|=====> o_hi_sub_sync  / o_hi_sub_last  (к pulser)
                      |                                       |
  i_adc_chan_ready -->| [3-Stage Synchronizer] -> chan_ready  |
     [3:0]            |                                       |
                      +---------------------------------------+
```

## 2. Тактовые домены и сбросы

Модуль осуществляет междоменную синхронизацию, поэтому к нему подключены все тактовые частоты и синхронные сбросы проекта.

| Имя тактовой частоты | Частота | Описание домена | Сброс (от `rst_sync`) |
| :--- | :--- | :--- | :--- |
| **sys_clk** | 80 MHz | Системный домен управления и FSM | `sys_rst_n` |
| **adc_clk** | 65 MHz | Домен оцифровки основного АЦП (VGA) | `adc_rst_n` |
| **log_clk** | 25 MHz | Домен оцифровки логарифмического АЦП | `log_rst_n` |
| **dac_clk** | 50 MHz | Домен ЦАП (генерация ВАРУ) | `dac_rst_n` |
| **hi_clk** | 250 MHz | Домен сверхбыстрого тайминга генераторов | `hi_rst_n` |

## 3. Порты модуля

```
                     +-----------------------+
                     |       sequencer       |
       sys_clk ----->|                       |=====> o_sys_sub_sync / o_sys_sub_last
       sys_rst_n --->|                       |=====> o_adc_sub_sync / o_adc_sub_last
                     |                       |=====> o_log_sub_sync / o_log_sub_last
       adc_clk ----->|                       |=====> o_dac_sub_sync / o_dac_sub_last
       adc_rst_n --->|                       |=====> o_hi_sub_sync  / o_hi_sub_last
                     |                       |
       log_clk ----->|                       |=====> o_step_index [1:0]
       log_rst_n --->|                       |=====> o_seq_busy
                     |                       |
       dac_clk ----->|                       |
       dac_rst_n --->|                       |
                     |                       |
       hi_clk ------>|                       |
      hi_rst_n ----->|                       |
                     |                       |
     i_sys_sync ---->|                       |
     i_seq_count --->| [1:0]                 |
  i_adc_chan_ready ->| [3:0] (from ascan)    |
                     +-----------------------+
```

### Управляющие интерфейсы
| Имя порта | Направление | Разрядность | Описание |
| :--- | :---: | :---: | :--- |
| **i_sys_sync** | вход | 1 | Сигнал глобального запуска кадра сканирования (одиночный импульс в домене `sys_clk`) |
| **i_seq_count** | вход | 2 | Количество активных шагов в кадре (00: 1 шаг, 01: 2 шага, 10: 3 шага, 11: 4 шага) |
| **i_adc_chan_ready** | вход | 4 | Сигналы готовности от физических модулей `ascan 0..3` (приходят в домене `adc_clk`) |

### Выходные синхроимпульсы запуска (CDC трансляция `sub_sync`)
| Имя порта | Направление | Разрядность | Описание |
| :--- | :---: | :---: | :--- |
| **o_sys_sub_sync** | выход | 1 | Импульс запуска субцикла в домене `sys_clk` (1 такт `sys_clk`) |
| **o_adc_sub_sync** | выход | 1 | Импульс запуска субцикла в домене `adc_clk` (1 такт `adc_clk`). Инициирует прием в `ascan` |
| **o_log_sub_sync** | выход | 1 | Импульс запуска субцикла в домене `log_clk` (1 такт `log_clk`). Инициирует логарифмический прием |
| **o_dac_sub_sync** | выход | 1 | Импульс запуска субцикла в домене `dac_clk` (1 такт `dac_clk`). Запускает генерацию профиля ВАРУ |
| **o_hi_sub_sync** | выход | 1 | Импульс запуска субцикла в домене `hi_clk` (1 такт `hi_clk`). Запускает генераторы `pulser` |

### Выходные синхроимпульсы признака последнего субцикла (CDC трансляция `sub_last`)
| Имя порта | Направление | Разрядность | Описание |
| :--- | :---: | :---: | :--- |
| **o_sys_sub_last** | выход | 1 | Импульс финального субцикла в домене `sys_clk` (1 такт `sys_clk`) |
| **o_adc_sub_last** | выход | 1 | Импульс финального субцикла в домене `adc_clk` (1 такт `adc_clk`). Используется в `ascan` для фиксации конца кадра |
| **o_log_sub_last** | выход | 1 | Импульс финального субцикла в домене `log_clk` (1 такт `log_clk`) |
| **o_dac_sub_last** | выход | 1 | Импульс финального субцикла в домене `dac_clk` (1 такт `dac_clk`) |
| **o_hi_sub_last** | выход | 1 | Импульс финального субцикла в домене `hi_clk` (1 такт `hi_clk`) |

### Выходные сигналы адресации и состояния
| Имя порта | Направление | Разрядность | Описание |
| :--- | :---: | :---: | :--- |
| **o_step_index** | output | 2 | Глобальный индекс текущего шага зондирования [0..3]. Считывается модулем `Param` для прямой адресации настроек каналов, масок и коммутаторов |
| **o_seq_busy** | output | 1 | Флаг выполнения кадра сканирования (лог. `1` от момента `i_sys_sync` до завершения последнего шага) |

## 4. Алгоритм работы и конечное состояние (FSM)

Управление циклом реализовано на базе синхронного автомата состояний в домене `sys_clk`.

### Описание состояний FSM

1. **`ST_IDLE` (Ожидание глобального запуска):**
   * Автомат находится в режиме ожидания. `o_seq_busy <= 0`, `step_index <= 0`.
   * При получении импульса `i_sys_sync == 1` происходит переход в `ST_INIT`.

2. **`ST_INIT` (Глобальная инициализация кадра):**
   * Активируется флаг занятости: `o_seq_busy <= 1`.
   * Сбрасывается индекс шагов: `step_index <= 0`.
   * Выдерживается пауза в 1 такт для гарантированного защелкивания параметров в модулях `Param`, `ascan` и `pulser`.
   * Переход в `ST_SETTLE`.

3. **`ST_SETTLE` (Установка параметров и стабилизация):**
   * Выставляется текущий индекс шага на шину: `o_step_index <= step_index`.
   * Модуль `Param` реагирует на изменение `o_step_index` и переключает аналоговые мультиплексоры ПЭП, а также обновляет маски каналов.
   * Выдерживается фиксированная пауза (2 такта `sys_clk`) для гарантированной стабилизации переходных процессов в аналоговых ключах.
   * Переход в `ST_TRIGGER`.

4. **`ST_TRIGGER` (Формирование импульсов запуска субцикла):**
   * Генерируется импульс запуска `o_sys_sub_sync <= 1`.
   * Если текущий индекс шага равен финальному значению (`step_index == i_seq_count`), генерируется импульс окончания последовательности: `o_sys_sub_last <= 1`.
   * Начинается автоматический процесс трансляции этих импульсов через схемы CDC во все целевые домены (`adc_clk`, `log_clk`, `dac_clk`, `hi_clk`).
   * Переход в `ST_WAIT_READY`.

5. **`ST_WAIT_READY` (Ожидание завершения регистрации):**
   * Сбрасываются пусковые сигналы: `o_sys_sub_sync <= 0`, `o_sys_sub_last <= 0`.
   * Автомат опрашивает выходы внутренних CDC-синхронизаторов для шины готовности.
   * Ожидание, пока все 4 синхронизированных флага готовности установятся в `1` (`synced_chan_ready == 4'b1111`).
   * После получения готовности:
     * Если `step_index == i_seq_count` $\rightarrow$ кадр сканирования полностью завершен, переход в `ST_IDLE`.
     * Иначе $\rightarrow$ `step_index <= step_index + 1`, переход в `ST_SETTLE`.

## 5. Временные ограничения и CDC-реализация

* **Когерентность запуска и конца кадра**: Сигналы `sub_sync` и `sub_last` транслируются через идентичные цепи CDC. Это гарантирует их строго одновременное появление во всех целевых тактовых доменах (в один и тот же физический такт АЦП/генератора).
* **Синхронизация входов готовности**: Сигналы готовности `i_adc_chan_ready[3:0]` от модулей `ascan`, работающих на частоте 65 МГц (`adc_clk`), переводятся в системный домен `sys_clk` (80 МГц) с помощью трехтриггерных цепочек синхронизации (`std_sync_stage`). Это предотвращает появление эффекта метастабильности в схеме сравнения автомата FSM.
* **Синхронизация импульсов через CDC**: Перенос импульсов `sub_sync` и `sub_last` из домена `sys_clk` во все тактовые домены (`adc_clk`, `log_clk`, `dac_clk`, `hi_clk`) реализован на базе схем "Toggle-Synchronizer-Edge Detector". Длительность результирующих импульсов составляет ровно один такт в каждом целевом домене.
* **Временная стабильность**: Изменение индекса шага `o_step_index` происходит строго в состоянии `ST_SETTLE` за 2 такта до выдачи пусковых импульсов. Это гарантирует, что к моменту прихода `o_hi_sub_sync` и `o_adc_sub_sync` все аналоговые переключения завершены, а новые цифровые параметры стабильно зафиксированы.
