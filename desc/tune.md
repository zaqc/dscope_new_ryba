# Описание модуля регулировки усиления и построения кривой ВРЧ (tune)

---

## 1. Общий обзор и назначение модуля

Модуль `tune` реализует прецизионное цифровое управление временной регулировкой чувствительности (ВРЧ / Time-Gain Compensation, TGC) в приемном тракте 16-канального ультразвукового дефектоскопа. 

Физический аналоговый тракт прибора содержит два последовательно включенных регулируемых напряжением усилителя (VGA) типа **AD604** (или два **AD603**), управляющие входы которых подключены к двум быстрым одноканальным 10-разрядным ЦАП **DAC101S101**. Дополнительно в приемном тракте присутствует логарифмический детектор **AD8310**, требующий индивидуальной подстройки аналогового смещения нуля, которая осуществляется с помощью второго аналогичного ЦАП.

Основная задача модуля `tune` — в каждом измерительном субцикле динамически вычислять и выдавать на ЦАП цифровые коды управления усилением так, чтобы компенсировать затухание ультразвуковой волны в объекте контроля по мере увеличения глубины (времени пробега импульса).

---

## 2. Параметры конфигурации модуля

Все параметры конфигурации подготавливаются в модуле `param` и поступают в модуль `tune` уже синхронизированными с исполнительным тактовым доменом `dac_clk`. Защелкивание параметров происходит по сигналу запуска `i_dac_sync`.

### 2.1. Верилог-параметры (Parameters)
| Имя параметра | Значение по умолчанию | Описание |
| :--- | :---: | :--- |
| `GAIN_FRAC_WIDTH` | `16` | Разрядность дробной части ($N$) для представления шага приращения (fixed-point) |

### 2.2. Входные порты конфигурации
| Имя параметра | Разрядность | Тактовый домен | Описание |
| :--- | :---: | :---: | :--- |
| `i_start_amp` | `[10:0]` | `dac_clk` | Начальное суммарное значение усиления при $t=0$ (непосредственно в кодах ЦАП, диапазон 0..2047) |
| `i_drop_ticks` | `[15:0]` | `dac_clk` | Время задержки начала нарастания ВРЧ в тактах `dac_clk` |
| `i_amp_one` | `[31:0]` | `dac_clk` | Приращение усиления на первом сегменте (умноженное на $2^{GAIN\_FRAC\_WIDTH}$) |
| `i_amp_two` | `[31:0]` | `dac_clk` | Приращение усиления на втором сегменте (умноженное на $2^{GAIN\_FRAC\_WIDTH}$) |
| `i_vrc_len` | `[15:0]` | `dac_clk` | Длина первого сегмента (точка перегиба) в тактах частоты `dac_clk` |
| `i_dac_min` | `[9:0]` | `dac_clk` | Минимально допустимый код для выдачи на ЦАП 1 и 2 |
| `i_dac_max` | `[9:0]` | `dac_clk` | Максимально допустимый код для выдачи на ЦАП 1 and 2 |
| `i_tune_mode` | `[1:0]` | `dac_clk` | Режим работы регулятора (последовательный / параллельный) |
| `i_log_offset` | `[9:0]` | `dac_clk` | Постоянное смещение для логарифмического тракта AD8310 |

## 3. Математическая модель построения кривой ВРЧ

Кривая ВРЧ представляет собой кусочно-линейную функцию (в логарифмическом масштабе дБ), которая начинает формироваться синхронно с запуском измерительного цикла по сигналу `i_dac_sync`, но имеет начальную фазу задержки (плато стабильного усиления).

```
 Усиление (дБ)
      ^
      |                                              _,' (приращение amp_two - пологое)
      |                                          _,-'
      |                                         * Точка перегиба
      |                                        /|
      |                                       / |
      |                                      /  | (приращение amp_one - крутое)
      |                                     /   |
      |____________________________________/    |
  start_amp             :                  :    |
      +-----------------+------------------+----+-------------------------> Время (такты dac_clk)
     t=0           t=drop_ticks            :    t=drop_ticks + vrc_len
                                           :<------ vrc_len ------>|
```

### 3.1. Использование фиксированной запятой (Q-формат)
Поскольку шаг приращения на один такт частоты `dac_clk` (50 МГц) может быть чрезвычайно малым, параметры приращений `i_amp_one` и `i_amp_two` передаются как целочисленные значения с фиксированной запятой, где разрядность дробной части задается параметром `GAIN_FRAC_WIDTH` (обозначим как $N$).

Связь между физическим приращением за такт $Step_{float}$ и кодом параметра определяется как:
$$i\_amp = \text{round}(Step_{float} \times 2^{GAIN\_FRAC\_WIDTH})$$

Параметр `i_start_amp` представляет собой сквозное суммарное управляющее значение для обоих каскадов и передается непосредственно как целое число в диапазоне от `0` до `2047` (где значение `2046` соответствует полной шкале двух последовательных ЦАП: $1023 + 1023$) без предварительного умножения на стороне хоста.

### 3.2. Алгоритм вычисления текущего усиления
1. **Инициализация:** По импульсу запуска `i_dac_sync` внутренний счетчик времени `r_time_cnt` сбрасывается в `0`, а 32-разрядный накопительный аккумулятор общего усиления `r_gain_acc` инициализируется стартовым значением `i_start_amp`, выровненным по сетке фиксированной запятой за счет сдвига влево:
   $$r\_gain\_acc(0) = i\_start\_amp \times 2^{GAIN\_FRAC\_WIDTH} = i\_start\_amp \ll GAIN\_FRAC\_WIDTH$$
   $$r\_time\_cnt(0) = 0$$

2. **Инкрементирование счетчика и аккумулятора (на каждом такте `dac_clk`):**
   На каждом такте (если не активен сброс) `r_time_cnt` увеличивается на 1. Вычисление аккумулятора производится по следующим правилам:
   
   * **Фаза 1: Задержка ($r\_time\_cnt \le i\_drop\_ticks$):**
     Аккумулятор не изменяет своего значения, на выходе сохраняется стартовое усиление:
     $$r\_gain\_acc(t) = r\_gain\_acc(t-1)$$
     
   * **Фаза 2: Первый сегмент ($i\_drop\_ticks < r\_time\_cnt \le (i\_drop\_ticks + i\_vrc\_len)$):**
     Длится ровно `i_vrc_len` тактов. К аккумулятору прибавляется шаг первого сегмента (крутой наклон):
     $$r\_gain\_acc(t) = r\_gain\_acc(t-1) + i\_amp\_one$$
     
   * **Фаза 3: Второй сегмент ($r\_time\_cnt > (i\_drop\_ticks + i\_vrc\_len)$):**
     К аккумулятору прибавляется шаг второго сегмента (пологий наклон):
     $$r\_gain\_acc(t) = r\_gain\_acc(t-1) + i\_amp\_two$$

3. **Извлечение целой части:** Для дальнейших расчетов канальных кодов используется целая часть накопленного усиления $Gain(t)$, получаемая путем сдвига аккумулятора вправо:
   $$Gain(t) = r\_gain\_acc(t) \gg GAIN\_FRAC\_WIDTH$$

## 4. Режимы регулировки каскадов усиления (`i_tune_mode`)

Поскольку два усилителя VGA включены последовательно, суммарное физическое усиление тракта (в дБ) является суммой усилений каждого из каскадов:

$$Gain_{total} = Gain_{stage1} + Gain_{stage2}$$

Модуль производит распределение расчетного значения общего усиления $Gain(t)$ на каналы управления ЦАП 1 (`o_dac_one`) и ЦАП 2 (`o_dac_two`) в зависимости от выбранного режима.

### А. Последовательный режим (`i_tune_mode = 2'b00`)
Данный режим обеспечивает минимальный уровень шумов на малых значениях общего усиления. Регулировка каскадов происходит строго поочередно:
1. Пока $Gain(t)$ меньше максимального порога одного каскада, регулируется только первый каскад. Второй каскад удерживается на минимуме:
   $$\text{Если } (Gain(t) - i\_dac\_min) < (i\_dac\_max - i\_dac\_min):$$
   $$o\_dac\_one(t) = Gain(t)$$
   $$o\_dac\_two(t) = i\_dac\_min$$

2. При превышении порога первый каскад фиксируется на максимуме, а регулировка переносится на второй каскад:
   $$\text{Если } (Gain(t) - i\_dac\_min) \ge (i\_dac\_max - i\_dac\_min):$$
   $$o\_dac\_one(t) = i\_dac\_max$$
   $$o\_dac\_two(t) = Gain(t) - (i\_dac\_max - i\_dac\_min)$$

Каждый из выходов дополнительно ограничивается снизу параметром `i_dac_min` и сверху `i_dac_max`.

```
  Код ЦАП
    ^
dac_max |             /---------- (ЦАП 1 - Первый каскад)
        |            /
        |           /   /-------- (ЦАП 2 - Второй каскад)
        |          /   /
dac_min |_________/___/__________
        +---------+---+-----------> Текущее расчетное усиление Gain(t)
```

### Б. Параллельный режим (`i_tune_mode = 2'b01`)
В этом режиме регулировка обоих каскадов происходит синхронно и синфазно. 

При переключении в параллельный режим **метод вычисления общего усиления $Gain(t)$ в аккумуляторе не меняется**: аккумулятор инициализируется тем же значением `i_start_amp` (до 2047) и накапливает шаги по абсолютно идентичному алгоритму. 

Однако, так как регулирующее напряжение прикладывается одновременно к двум последовательным каскадам, итоговое усиление в децибелах нарастает в 2 раза быстрее. Для сохранения заданного наклона кривой вычисленный код общего усиления $Gain(t)$ делится на 2 (сдвиг вправо на 1 бит):

$$o\_dac\_one(t) = o\_dac\_two(t) = \frac{Gain(t)}{2} = Gain(t) \gg 1$$

После деления оба канала индивидуально ограничиваются («насыщаются») по порогам `i_dac_min` и `i_dac_max`:

```verilog
// Пример логики ограничения в параллельном режиме
wire [10:0] gain_div2 = Gain(t) >> 1;

always @(*) begin
    o_dac_one = (gain_div2 < i_dac_min) ? i_dac_min : 
                (gain_div2 > i_dac_max) ? i_dac_max : gain_div2[9:0];
    o_dac_two = o_dac_one;
end
```

## 5. Коррекция логарифмического тракта (Offset)

Канал логарифмического детектора AD8310 имеет статическую калибровку смещения нуля. Модуль принимает параметр `i_log_offset` `[9:0]` в домене `dac_clk` и напрямую транслирует его на выходной порт `o_dac_offset` для непрерывной выдачи в контроллер третьего ЦАП. Это значение остается неизменным в течение всего измерительного цикла и обновляется только при изменении конфигурации хостом.

## 6. Порты ввода-вывода модуля `tune`

```verilog
module tune #(
    parameter GAIN_FRAC_WIDTH = 16     // Разрядность дробной части для фиксированной запятой
)(
    // Тактирование и синхронные сбросы (без префикса i_, суффикс _n для сброса)
    input  wire        dac_clk,        // Клок ЦАП ВРЧ 50 МГц (генерация кривой)
    input  wire        dac_rst_n,      // Синхронный сброс домена dac_clk

    // Синхронизация запуска (длительность - 1 такт соответствующего домена)
    input  wire        i_dac_sync,     // Строб начала генерации ВРЧ (домен dac_clk)

    // Параметры конфигурации кривой (защелкнуты из модуля param в домене dac_clk)
    input  wire [10:0] i_start_amp,    // Начальное суммарное значение общего усиления в кодах ЦАП (0..2047)
    input  wire [15:0] i_drop_ticks,   // Количество тактов задержки перед нарастанием ВРЧ
    input  wire [31:0] i_amp_one,      // Шаг приращения на 1-м сегменте (умноженный на 2^GAIN_FRAC_WIDTH)
    input  wire [31:0] i_amp_two,      // Шаг приращения на 2-м сегменте (умноженный на 2^GAIN_FRAC_WIDTH)
    input  wire [15:0] i_vrc_len,      // Длительность 1-го сегмента в тактах dac_clk
    input  wire [9:0]  i_dac_min,      // Минимальное ограничение кода ЦАП
    input  wire [9:0]  i_dac_max,      // Максимальное ограничение кода ЦАП
    input  wire [1:0]  i_tune_mode,    // Режим регулировки (последовательный/параллельный)
    input  wire [9:0]  i_log_offset,   // Статическое смещение логарифмического тракта

    // Интерфейс выдачи данных на SPI передатчики ЦАП
    output reg  [9:0]  o_dac_one,      // Значение для ЦАП первого каскада VGA
    output reg  [9:0]  o_dac_two,      // Значение для ЦАП второго каскада VGA
    output reg  [9:0]  o_dac_offset,   // Значение для ЦАП смещения логарифмического усилителя
    output reg         o_dac_data_vld  // Строб валидности новых кодов для SPI-передатчика
);
